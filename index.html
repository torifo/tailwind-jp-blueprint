ダークモード戦略

darkMode: 'class' を採用。 JavaScript から html 要素に .dark を付け外しするだけで全体が切り替わります。

// tailwind.config.js darkMode: 'class', // global.css — 変数の上書きのみでテーマ切り替えが完結 .dark { --color-bg-base: #0f172a; --color-bg-sunken: #020617; --color-text-main: #f8fafc; --color-text-muted: #94a3b8; } // JS — クラスの付け外しだけでOK document.documentElement.classList.toggle('dark');

section 02

意図的に
形式化しない領域

すべてを config に詰め込むとファイルが肥大化し、柔軟性が失われます。 以下の要素は「その場のコード」で書くべき領域として意図的に残しています。形式化しないことが表現力を守ります。

一過性のレイアウト調整 任意値で記述

特定の場所だけで必要な微調整。Tailwind の任意値（Arbitrary Values）構文でその場に書きます。 config に登録すると、再利用されない値でファイルが汚れるうえ、他のパーツにも波及するリスクがあります。

<!-- その場で書く ✓ --> <div class="top-[13px] left-[5%] w-[calc(100%-2rem)]"> <div class="mt-[3px] gap-[7px] translate-x-[2px]"> <!-- config に追加しない ✗ --> // 一回しか使わない spacing をグローバル定義しない
複雑なグラデーション インラインまたはユーティリティ組み合わせ

方向・色数・透明度が複雑に組み合わさるグラデーションは、ユーティリティの直接組み合わせか インラインスタイルの方が直感的です。セマンティクスよりビジュアルの正確さが優先されます。

<!-- ユーティリティ組み合わせ ✓ --> <div class="bg-gradient-to-br from-blue-500/20 via-purple-500/10 to-transparent"> <!-- 複雑な場合はインラインスタイル ✓ --> <div style="background: radial-gradient(circle at 30% 40%, #3b82f620, transparent 60%)">
特定ページ専用の装飾 コンポーネント内で完結

キャンペーンバナー・LP のヒーロー装飾・期間限定プロモーション要素など、全体で使い回さないパーツは 標準化の対象外です。必要な場所にそのまま書き、デザインシステムを汚染しません。

強い世界観を持つテーマデザイン テーマ専用の追加実装

Tailwind は「道具箱」であり「作風」ではありません。手書き風・コミック風などの世界観は Tailwind 単体では実現できず、別途追加実装が必要です。

テーマ Tailwind だけでは無理な部分 実装アプローチ
手書き風 完全に不規則な境界線・ガタガタした線 SVG フィルター + 歪んだ borderRadius の多用
コミック風 極太の枠線・二重オフセット影・スクリーントーン背景 box-shadow の多重定義 + カスタム背景画像
共通 独特なフォント（世界観の7割を決める） Google Fonts から font-display として読み込み Config で紐付け

なぜ形式化しないか： 手書き風の美しさは「不規則であること」が源泉です。config で「標準の角丸」として定義すると、 すべてのパーツが同じ歪み方になり、デジタルで不自然な印象になります。

推奨アプローチ： 「土台はモダンな Tailwind 設定で作りつつ、テーマパーツだけを自由に書き込む」スタイルが最も表現力が高くなります。

メンテナンスの指針

「何を変えたい時に、どこを触るか」を3つのルールで整理します。 このシステムに沿って運用すれば、変更の影響範囲が常に予測可能になります。

01
色を変えたい
CSS変数の値を書き換えるだけで、全コンポーネントに伝播します。コンポーネントを一つ一つ直す必要はありません。
global.css
02
新しいフォントを追加したい
fontFamily オブジェクトにキーを追加するだけでクラス名が自動生成され、font-* として使えます。
tailwind.config.js
03
新しいアニメーションを作りたい
汎用的なら config へ登録。その場限りなら animate-[...] でインライン記述します。
config or HTML

  • example/global.css — CSS変数の実値（ライト／ダークモード対応）
